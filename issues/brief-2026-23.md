## 5. AI Security Risks & Real-World Attacks (SDC core)

- **Sysdig: first confirmed in-the-wild fully autonomous LLM-agent intrusion — reported May 30, 2026 (incident May 10).** Attacker chained CVE-2026-39987 (Marimo RCE) → an LLM agent then autonomously did 4 privilege-escalation pivots (AWS Secrets Manager, SSH bastion) and exfiltrated an internal PostgreSQL DB in <2 min; full chain <1 hr. Agent ran 12 API calls across 11 IPs in 22s via Cloudflare Workers egress pool to beat per-IP rate limits; adapted in real time. https://www.sysdig.com/blog/ai-agent-at-the-wheel-how-an-attacker-used-llms-to-move-from-a-cve-to-an-internal-database-in-4-pivots ; The Hacker News 2026-05-30.
- **Prompt-injection → RCE in AI agent frameworks (theme):**
  - Langroid SQLChatAgent — CVE-2026-25879, CVSS 9.8 — June 1, 2026. Injected input coerces LLM into arbitrary SQL → RCE where DB role has code-exec privileges. Fixed in 0.63.0 (SELECT-only default). TheHackerWire.
  - Windsurf MCP — CVE-2026-30615 — zero-click: malicious HTML auto-registers attacker-controlled MCP STDIO server → command exec. SentinelOne; OX Security. NOTE: some write-ups frame this as a flaw in MCP's trust model broadly — contested; the defect is in Windsurf's input validation. [REVIEW framing + confirm exact date]
  - Microsoft Semantic Kernel — CVE-2026-26030 / CVE-2026-25592 — RCE / arbitrary file write via functions exposed to the model (Microsoft Security Blog, May 7). Pre-window but part of the pattern.
- **AI software-supply-chain:**
  - "Miasma" npm worm via phantom `binding.gyp` (node-gyp auto-rebuild, no postinstall needed) — June 3–5, 2026. 57 packages / 286+ versions in ~2 hrs; biggest hit @vapi-ai/server-sdk (voice-AI SDK, 408k+ monthly downloads); also ai-sdk-ollama, RedHat packages (RedHat-32 hit June 1). Harvests npm/GitHub/AWS/GCP/Azure/Vault/K8s creds; self-propagates. StepSecurity; Corgea.
  - OpenAI macOS code-signing cert revocation (TanStack "Mini Shai-Hulud" compromise, two OpenAI dev devices affected) — deadline June 12, 2026 for ChatGPT Desktop / Codex / Atlas users to update. https://openai.com/index/our-response-to-the-tanstack-npm-supply-chain-attack/
- **AI as offense + defense:** Xint Code (autonomous AI security tool) found a 2-yr-old Redis use-after-free RCE — CVE-2026-23479 (CVSS 8.8) — patched/disclosed June 3, 2026, present 7.2.x–8.6.x. The Hacker News.
